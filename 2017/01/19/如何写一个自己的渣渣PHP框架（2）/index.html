既然如此，就先把自动加载说一下。

框架来说，简单点，就是根据你的ur地址，比如 index.php?c=index&a=run 大多数应该都是这样吧，那么框架是如何根据 index 和 run 去找到 index 控制器，然后调用其中的 run 方法。当然后面还可以加很多的参数暂且先不说。确切的来讲你的 index 控制器也是一个类，你什么时候 new 的这个类呢？你什么时候调用的 run 方法？这就是框架要做的事情，就是我理解的 路由 和 自动加载 。

接着来，建立框架核心目录 /data/momoma/core 在core目录建立 init.php 文件作为框架核心入口文件。 建立目录 /data/momoma/core/momoma 在 momoma 里建立 AutoLoad.php 文件，（文字表述写框架的过程简直是好无力呀）：

/**
 * 自动加载类
 * User: momoma
 * Date: 2017/1/19
 * Time: 上午10:41
 */
namespace core\momoma;

class AutoLoad {

    public static function loadprint($class){
        $file = MOMOMA_PATH."/".$class.".php";
        // "Linux需要转义斜杠";
        $file = str_replace('\\','/',$file);
        echo $file."<br>";    //这句话是为了调试使用
        if (is_file($file)) {
            include($file);
        }
    }
}

使用命名空间，你使用过命名空间吗？没有的话一定要用上。你说这么小的项目完全不需要使用命名空间啊，小项目正好用熟悉一下，不然大项目更不会用。(命名空间一定要根据目录来定义，这样为自动加载做准备)

然后在 momoma 里建立 Momoma.php 文件，作为框架核心类。

/**
 * 框架核心类
 * User: momoma
 * Date: 2017/1/19
 * Time: 上午10:52
 */

namespace core\momoma;

class Momoma {

    public function run(){
        echo "框架核心入口";
    }
}

这个时候来看 init.php :

/**
 * 框架核心入口文件
 * User: momoma
 * Date: 2017/1/19
 * Time: 上午10:39
 */

require_once MOMOMA_PATH.'/core/momoma/AutoLoad.php';

use \core\momoma\AutoLoad;
use \core\momoma\Momoma;

// 自动加载
spl_autoload_register([new AutoLoad(),'loadprint']);

// 框架核心入口
$momoma = new Momoma();
$momoma->run();

spl_autoload_register 是今天的重点。

看了半天php官方文档，说实话我还是没有理解其中的意思。我自己的理解就是，当你要在代码中 new 一个 Route 类的时候，你肯定得先 include 这个类文件。不然会直接报错告诉你 “Class ‘Route’ not found” 的。比如我在上面 new 了 Momoma 这个类，我没有 include ，这个时候 spl_autoload_register 就会发生作用，当我要 new 一个不存在的类的时候，程序就会自动触发 spl_autoload_register 自动注册的方法，也就是 AutoLoad 类里面的 loadprint 方法。loadprint 方法会接收到你要 new 的这个类，包含命名空间。然后，定义这个文件，$file = MOMOMA_PATH."/".$class.".php"; 转义，然后判断它是否是一个文件，是就 include 。

  • 这里我们做一个调试，也就是上面 AutoLoad 里的这句话，echo $file."
    "; //这句话是为了调试使用     我们把自动加载的文件路径打印出来。
